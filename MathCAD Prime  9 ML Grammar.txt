Mathcad Prime 9 – Math Region Markup Language (ML)
Corrected Grammar Notes and Practical Rules
================================================

PURPOSE
-------
This document captures empirically verified Mathcad Prime ML grammar rules,
with emphasis on matrix content, identifiers, subscripts, Greek symbols,
and unit expressions. These rules are based on known‑good paste behavior
in Mathcad Prime and override earlier incorrect assumptions.

These notes MUST be followed when generating ML intended for direct
paste into Mathcad Prime.


CORE PRINCIPLE
--------------
Inside Mathcad ML matrices, identifiers are NOT declared variables.
They must be represented as raw identifiers or symbols, not wrapped
in @LABEL VARIABLE.


IDENTIFIERS AND VARIABLES
-------------------------

1. Bare identifiers are valid inside matrices:
   - I
   - V
   - ω
   - Δθ

2. Subscripted identifiers MUST use @ID and @SUB:
   Example:
     (@ID R (@SUB ac))
     (@ID W (@SUB c))
     (@ID θ (@SUB a))
     (@ID λ (@SUB 1))

3. DO NOT wrap identifiers inside matrices with:
     (@LABEL VARIABLE ...)
   This will cause Mathcad paste failure.

   INCORRECT:
     (@LABEL VARIABLE (@ID R (@SUB ac)))

   CORRECT:
     (@ID R (@SUB ac))


GREEK SYMBOLS
------------

Greek symbols may appear either:
- Bare: ω, Δθ
- As @ID bases with subscripts: (@ID θ (@SUB c))

No VARIABLE label wrapper is required or allowed inside matrices.


MATRICES
--------

Matrix form:
  (@MATRIX rows cols element1 element2 element3 ...)

Rules:
- Elements are positional, not typed
- Each row must contain exactly <cols> elements
- Matrices commonly used for symbol / description / unit tables


UNITS – CRITICAL RULES
---------------------

1. Only base units may appear as @LABEL UNIT:
   Examples:
     (@LABEL UNIT A)
     (@LABEL UNIT V)
     (@LABEL UNIT W)
     (@LABEL UNIT m)
     (@LABEL UNIT K)
     (@LABEL UNIT degC)
     (@LABEL UNIT Ω)
     (@LABEL UNIT rad)
     (@LABEL UNIT s)

2. Compound units MUST be expressed as unit expressions.
   NEVER use composite unit strings.

   INCORRECT:
     (@LABEL UNIT W/m)
     (@LABEL UNIT Ω/m)
     (@LABEL UNIT rad/s)
     (@LABEL UNIT W/(m·K))

   CORRECT:
     (/ (@LABEL UNIT W) (@LABEL UNIT m))
     (/ (@LABEL UNIT Ω) (@LABEL UNIT m))
     (/ (@LABEL UNIT rad) (@LABEL UNIT s))
     (/ (@LABEL UNIT W) (* (@LABEL UNIT m) (@LABEL UNIT K)))

3. Thermal resistance per unit length (K·m/W):
   Express explicitly as:
     (* (@LABEL UNIT K) (/ (@LABEL UNIT m) (@LABEL UNIT W)))


TEMPERATURE UNITS
-----------------

Use:
  (@LABEL UNIT degC)

Do NOT use:
  °C
  (@LABEL UNIT °C)


DIMENSIONLESS QUANTITIES
------------------------

Dimensionless values MUST be represented as literal 1.

Example:
  λ₁, λ₂ ratios:
    1

Do NOT attempt to define a "unitless unit".


EXAMPLE – VALID MATRIX ROW
--------------------------

(@ID R (@SUB ac))
"AC resistance per unit length"
(/ (@LABEL UNIT Ω) (@LABEL UNIT m))


COMMON FAILURE MODES (DO NOT DO THESE)
-------------------------------------

- Wrapping identifiers in @LABEL VARIABLE inside matrices
- Using composite unit strings (e.g., "W/m")
- Using °C instead of degC
- Omitting @ID/@SUB for subscripts
- Using @PLACEHOLDER tokens
- Mixing presentation notation with AST grammar


TOP‑LEVEL DEFINITIONS (OUTSIDE MATRICES)
----------------------------------------

Outside matrices (e.g., in definition regions), variables MAY be declared
using:

  (:= (@LABEL VARIABLE x) 5)

But this rule does NOT apply inside @MATRIX constructs.


SUMMARY (TL;DR)
---------------

Inside matrices:
- Identifiers are raw symbols or @ID nodes
- No VARIABLE labels
- All compound units are expressions
- degC only
- Dimensionless = 1

If Mathcad Prime can paste it, the grammar is correct.
If it cannot, assume one of the above rules was violated.
